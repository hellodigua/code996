# 任务：合并历史 Web 实现并设为默认报告

> 完成日期：2026-07-18 · 状态：已完成 · 类型：架构与用户体验升级

## 背景

历史仓库 `code996-web` 已实现一版可视化界面，但与当前 CLI 分离。该仓库计划归档，因此本次将可复用视觉、字体和图表组件迁入主仓库，并让 CLI 直接生成本地报告。

## 已确认决策

- 历史来源固定为 `code996-web@8abe74b7dadb9e16f1d0d8dc3d49f80b491f1c8e`。
- Web 源码进入主仓库 `web/`，发布产物进入 `dist/web/`，不再依赖旧仓库。
- CLI 计算一次并输出稳定 `ReportData`，Web 不复制分析算法。
- 默认打开本地文件，不启动常驻 localhost 服务。
- 报告默认保存到 `Downloads/code996-report/YYYY-MM-DD_HH-mm-ss_项目名/`，按名称排序即可找到最新报告，由用户自行删除；Downloads 不可写时提示并降级到系统临时目录。
- 页面支持中文和英文；开源项目、失败仓库和少于 50 个提交的样本不显示误导性评分。
- Web 必须是完整报告而非摘要页，CLI 已计算的证据应通过 `ReportData` 进入页面。
- 首屏尽量还原历史结果页的顶栏、方形指数卡和四行摘要；原站“返回”入口改为本地报告可用的“详细报告”锚点。
- 用户可通过 `--terminal` 保留传统体验，通过 `--no-open` 阻止自动打开。

## 交付

- Vue 3 + Vite 离线 Web 应用、像素终端视觉与响应式布局。
- 单仓库、多仓库、失败与低样本状态。
- CLI 同等信息密度的诊断、工时依据、项目分类、跨时区、加班频率、完整趋势与团队分析。
- 历史结果页风格首屏，包含仓库上下文、指数、推测工时、加班比例、提交数、分析时间和中英文详情跳转。
- CLI 输出模式自动判断、本地 HTML 生成、跨平台打开和失败降级。
- 可读且不会覆盖的报告目录命名，以及面向测试的输出根目录和时间注入能力。
- 中英文 README、架构文档、技术栈和路线图同步。

## 验证

- Jest 与 Vitest 全量测试。
- `npm run build` 同时生成 CLI 与 Web。
- 使用真实仓库执行 `code996 --web --no-open --all-time`，确认生成独立 HTML。
- 桌面和 390px 移动端视觉验收，无页面级横向溢出。
- 使用 ChatLab 的真实 `npm link` 报告验证项目分类维度、13 个月趋势明细和本地 HTML 数据注入。
- `npm audit --omit=dev` 与完整 `npm audit` 均为 0。
- `npm pack --dry-run` 检查 Web 产物随包发布、Web 源码不进入发布包。
