# 本地 Web 报告

## 目标

用户显式运行 `code996 --web` 后，看到由本次分析数据生成的本地网页。页面复用历史 `code996-web` 的像素终端视觉，但数据来自当前 CLI 的统一计算结果，不再维护第二套分析逻辑。

## 用户行为

- 默认输出传统终端报告，在 SSH、CI 和 AI 编程工具中也不会意外创建文件或打开浏览器。
- `--web`：生成并打开本地 Web；`--no-open`：仅生成并打印 HTML 路径。
- `--json`、`--md`、`--web` 互斥；JSON/Markdown 的 `--output` 行为保持不变。

## 架构

1. CLI 完成 Git 采集、工时推断、项目分类、趋势和团队分析。
2. `src/cli/output/json-formatter.ts` 构建语言中性的 `ReportData`。
3. `src/cli/output/output-mode.ts` 根据显式参数、TTY 和 CI 判断渲染器。
4. `src/cli/output/web-report-writer.ts` 复制 `dist/web` 到 `Downloads/code996-report/日期_时间_项目名/`，内联模块脚本并注入 ReportData。
5. `web/src/App.vue` 组织单仓库、多仓库、开源项目、失败仓库和低样本状态。
6. `web/src/components/DiagnosticInsights.vue`、`ClassificationEvidence.vue`、`TimezoneEvidence.vue`、`TrendAnalysis.vue`、`TeamAnalysis.vue` 分别渲染 CLI 的诊断与完整证据。

`ReportData` 的正式 Schema 位于 `src/report/report-data.ts`。该文件必须保持浏览器安全，不能引用 Node.js API 或 CLI 国际化模块。星期、项目类型、评分键等字段保持语言中性，中文和英文只在 Web 渲染层转换。

## 报告覆盖范围

Web 是完整报告而不是终端摘要，必须覆盖 CLI 已计算的主要证据：

- 核心指数、评级、加班比例、提交数以及与 CLI 相同阈值生成的风险结论。
- 工时起止点、候选区间、检测方法、样本量、置信度和超过 9 小时的计算提示。
- 项目分类的规律性、周末活动、业余时段、贡献者和四项规律性信号。
- 主导/非主导时区、睡眠窗口、置信度及各时区提交分布。
- 小时/星期分布、工作日加班、周末工作类型和夜间活动的周均/月均频率。
- 月度趋势的完整 10 列数据、标准差、汇总指标和质量说明。
- 团队首末提交分位数、指数强度分布、健康度、风险提示及贡献者完整明细。
- 多仓库的状态、项目类型、指数、加班比例、提交/贡献者数量和首末提交日期。

项目分类说明和团队健康结论不能直接复用 CLI 已翻译字符串。`ReportData` 只保存数值、枚举和布尔信号，Web 根据当前语言现场生成文案，确保页面切换语言后没有残留中文或英文。

## 页面结构与交互

- 首屏延续历史 `code996-web` 结果页：灰色顶栏、`#CODE996 Result` 像素标题、粉色方形指数卡，以及工时、加班比例、提交数和分析时间四项摘要。
- 桌面端主内容保持历史版本约 70% 的页面宽度，移动端切换为 90%；指数卡和摘要在窄屏纵向排列，详细报告中的宽表只在自身容器内滚动。
- 顶栏“详细报告”和指数说明中的链接都跳转到 `#report-details`。本地报告没有历史站点的输入页，因此不保留无法正确工作的“返回”按钮。
- 中文/英文切换与 GitHub 角标沿用历史位置和悬停反馈；切换语言只改变展示文案，不重新计算或丢失报告数据。
- 页面使用平滑锚点滚动；用户启用 `prefers-reduced-motion` 时关闭滚动和按钮动画。
- 开源项目与低样本报告仍使用相同顶栏和仓库上下文，但不渲染误导性的指数卡，改为展示适用性说明。

## 离线与生命周期

- Vite 使用相对资产路径构建，字体、favicon、CSS 和 JS 全部位于 `dist/web`。
- 生成时模块脚本内联到 HTML，避免浏览器对 `file://` 外部 ES Module 的限制。
- CLI 不启动 HTTP 服务；报告默认写入用户目录下的 `Downloads/code996-report/`，进程退出后仍可访问。
- 单次报告目录格式为 `YYYY-MM-DD_HH-mm-ss_项目名`，按名称排序即可按生成时间排列；同一项目在同一秒重复生成时追加 `_2`、`_3`，不覆盖旧报告。
- 项目名会移除跨平台非法文件名字符；多仓库报告使用 `<数量>-repos`。
- 用户删除单次报告目录即可同时移除 HTML、样式、字体和脚本。Downloads 不可写时降级到系统临时目录，并在终端明确提示。
- `tempRoot` 和 `now` 仅用于测试隔离与确定性断言，不是公开 CLI 参数。

## 安全与隐私

- 数据不上传，不加载 CDN，也不进行网络请求。
- 注入 HTML 前转义 `&`、`<`、`>`、U+2028 和 U+2029，防止仓库路径或错误信息闭合 `<script>`。
- macOS、Windows、Linux 分别使用 `open`、`explorer.exe`、`xdg-open`；参数直接传给子进程，不经过 shell。
- 自动打开失败不删除报告，CLI 会打印路径供手动打开。

## 质量门槛

- CLI/Jest：输出模式矩阵、可读目录名、同秒冲突、资产复制、恶意字符串注入、保存位置降级、打开失败降级、Schema 中性化。
- Web/Vitest：历史风格顶栏、摘要字段、详情锚点、中英文切换、开源项目不评分、多仓库失败状态、低样本提示、完整趋势列、分类证据与团队健康分析。
- 视觉验收：桌面与 390px 移动端，页面级无横向溢出，支持 `prefers-reduced-motion`。
- 发布验收：`npm pack --dry-run` 必须包含 `dist/web`，且生产依赖 `npm audit --omit=dev` 为 0。
